## 1. 项目概述

### 1.1 项目简介
数仓代码自动生成工具是一个基于 Python 的自动化脚本生成系统，用于从 Excel 调研文档自动生成数据仓库各层代码，包括：
- **ODS 层**：建表 DDL、DataX 抽数脚本（Shell）
- **DWD 层**：建表 DDL、ETL 加工 Shell 脚本
- **DWS 层**：建表 DDL、ETL 加工 Shell 脚本

### 1.2 技术栈
- **语言**：Python 3.8+
- **依赖库**：pandas、openpyxl、dataclasses
- **目标引擎**：Inceptor (Hive)、DataX、Beeline
- **目标存储**：HDFS

### 1.4 整体数据流
```
表级调研.xlsx ──→ parse_table_survey() ──→ [TableMeta] ──┐
                                                          ├──→ generate_all_ods_ddl()      → DDL SQL
字段级调研.xlsx ─→ parse_field_survey() ──→ {表名: [FieldMeta]} ──┘
                                                          └──→ generate_all_ods_etl()      → ETL Shell

MAPPING.xlsx ────→ parse_mapping_sheet() ──→ [MappingSheet] ──→ BaseGenerator.generate_ddl() → DDL SQL
                                                               └──→ BaseGenerator.generate_etl() → ETL Shell
```

### 2.2 核心数据类（`tools/core/ir.py`）

```python
@dataclass
class TableMeta:
    src_sys: str              # 源系统英文名（如 "O32"）
    src_db_type: str          # 源数据库类型（如 "ORACLE"）
    src_schema: str           # 源 schema（如 "FMP"）
    src_table: str            # 源表名（如 "TUNITSTOCK"）
    src_table_cn: str         # 源表中文名
    ods_table: str            # ODS 表名
    load_strategy: str        # 加载策略：FULL / INCR
    incr_cond: str            # 增量条件字段名
    incr_cond_format: str     # 增量条件格式（如 yyyyMMdd）
    is_partition: str         # 是否分区
    storage_period: str       # 存储周期
    topic: str                # 主题
    table_rows: str           # 表行数估算
    is_reserved: str          # 是否保留（是/否）

@dataclass
class FieldMeta:
    src_sys: str              # 源系统英文名
    src_table: str            # 源表名
    src_table_cn: str         # 源表中文名
    ordinal: float            # 字段序号
    src_name: str             # 字段英文名
    src_name_cn: str          # 字段中文名
    src_type: str             # 字段原始类型
    hive_type: str            # 转换后的 Hive 类型
    nullable: str             # 是否允许为空 Y/N
    pk_flag: str              # 是否主键
    biz_pk_flag: str          # 是否业务主键
    fk_flag: str              # 是否外键
    default_val: str          # 默认值
    is_ods: str               # 是否入 ODS
    note: str                 # 备注

@dataclass
class MappingRule:
    tgt_name: str             # 目标字段英文名
    tgt_name_cn: str          # 目标字段中文名
    tgt_type: str             # 目标字段类型（Hive 类型）
    tgt_ordinal: float        # 序号
    is_pk: str                # 是否主键
    group_no: str             # 组别
    src_field_alias: str      # 源字段别名（如 T1.CUST_ID）
    src_table_alias: str      # 源表别名（如 T1）
    src_table_name: str       # 源表名
    src_table_cn: str         # 源表中文名
    src_field_name: str       # 源字段名
    join_type: str            # JOIN 方式（MAIN TABLE / LEFT JOIN / ...）
    join_cond: str            # 关联条件（ON ...）
    filter_cond: str          # 过滤条件
    note: str                 # 备注

@dataclass
class MappingSheet:
    tgt_table_cn: str         # 目标表中文名
    tgt_table: str            # 目标表名
    func_desc: str            # 功能描述
    partition_col: str        # 分区字段（默认 p_dt）
    param: str                # 参数（默认 p_batch_dt）
    frequency: str            # 频度（默认 D）
    tgt_table_type: str       # 目标表类型
    load_type: str            # 加载类型
    incr_logic: str           # 增量逻辑
    source_tables: [str]      # 源表列表
    mappings: [MappingRule]   # 映射规则列表
```

## 3. 数仓层级规范

### 3.1 层级定义

| 层级 | Schema | 说明 | 分区字段 | 系统字段 |
|------|--------|------|----------|---------|
| ODS | `ODS_XDAY_{系统}` | 操作数据层，保持源系统字段名 | `P_DT` | 无系统字段 |
| DWD | `DWDXDAY` | 明细数据层，按主题建模 | `P_DT` | SSYS, SRC_TAB, LD_TIME, MODIFY_TIME |
| DWS | `DWSXDAY` | 汇总数据层，面向业务主题 | `P_DT` | LD_TIME, MODIFY_TIME |

### 3.2 表名规范

#### 3.2.1 ODS 表名
```
ODS_{系统}_{源表名}_{后缀}
```

| 系统 | 示例表名 | 说明 |
|------|---------|------|
| O32 | `ODS_O32_TUNITSTOCK_PFD` | 全量表 |
| HSZTA | `ODS_HSZTA_SCOMBI_PTD` | 增量表 |
| HSFA | `ODS_HSFA_CUST_INFO_PFD` | 全量表 |
| HSFTA | `ODS_HSFTA_ACCOUNT_PFD` | 全量表 |
| HSDS | `ODS_HSDS_CUSTOMER_PFD` | 全量表 |
| WIND | `ODS_WIND_INDEX_PFD` | 全量表 |
| JY | `ODS_JY_MARKET_PFD` | 全量表 |
| OFFW | `ODS_OFFW_NEWS_PFD` | 全量表 |
| OA | `ODS_OA_APPROVAL_PFD` | 全量表 |

#### 3.2.2 DWD 表名
```
DWD_{主题}_{实体}_{后缀}
```

**示例：**
- `DWD_AST_CRSE_INFO_PFD`
- `DWD_AST_CUST_HLDP_PTD`
- `DWD_PROD_INFO_PFD`

#### 3.2.3 DWS 表名
```
DWS_{主题}_{实体}_{后缀}
```

**示例：**
- `DWS_PROD_SCR_HLDP_DETAIL_PTD`
- `DWS_TRAN_ACCT_OPEN_BTCH_PFD`
- `DWS_CUST_HOLD_AST_SUM_PFD`

### 3.3 后缀规则

| 后缀 | 含义 | 说明 |
|------|------|------|
| `_PFD` | Partition Full Data | 全量数据，每日覆盖 |
| `_PTD` | Partition Time-slice Data | 增量数据，按时间切片 |

**判断依据：**
- 加载策略列包含"增量" → `_PTD`
- 加载策略为"全量" → `_PFD`

### 3.4 分区规范

**所有层统一使用：**
```sql
PARTITIONED BY (P_DT STRING)
```

**全量表（覆盖写入）：**
```sql
INSERT OVERWRITE TABLE schema.table PARTITION(P_DT='${p_end_dt}') ...
```

**增量表（动态分区）：**
```sql
set hive.exec.dynamic.partition=true;
set hive.exec.dynamic.partition.mode=nonstrict;
INSERT OVERWRITE TABLE schema.table PARTITION(P_DT) ...
```

### 3.5 存储格式
```sql
ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t' NULL DEFINED AS '';
```

## 4. Excel 调研文档格式规范

#### 4.1.1 文件命名

**示例：**

| 10 | 是否保留 | String | 是 | - | **是/否/保留** |

### 5.4 系统识别规则

- `02-O32/` → 提取 `O32` → 保持
- `03-HSFA/` → 提取 `HSFA` → 保持

**DWS 系统识别（从文件名提取）：**

### 5.5 过滤规则

- Sheet 名为 `目录`、`变更记录`、`修改记录`、`数据测试` → 跳过
- 以 `~$` 开头的文件（Excel 临时文件）→ 跳过
- "目标字段英文名"为空的数据行 → 跳过

### 5.6 字段名自动修复

非法字符（非字母数字下划线）自动替换：
```python
'OUT_VAL APPRE'     → 'OUT_VAL_APPRE'
'ORI_CRRC_VAL APPRE' → 'ORI_CRRC_VAL_APPRE'
'VAL APPRE_OC'      → 'VAL_APPRE_OC'
```

只应用于 `目标字段英文名`，不影响其他字段。

### 5.7 JOIN 解析

生成 FROM/JOIN 子句时：
- 别名标记为 `MAIN TABLE` → 主表，放在第一行
- 其他别名 → 按 `JOIN方式`（LEFT JOIN / RIGHT JOIN）拼接
- 关联条件 `ON` → 第一个条件跟在 ON 后面，后续 AND 换行缩进

```sql
DROP TABLE IF EXISTS ODS_XDAY_{系统}.ODS_{系统}_{源表}_{后缀};
```

### 6.2 DWD/DWS 层代码

#### 6.2.1 DDL 生成（`tools/generator/base.py`）

**生成逻辑：**
1. 目标字段名去重（防止重复列）
2. 字段按 `tgt_ordinal` 排序
3. 追加系统字段（DWD: SSYS, SRC_TAB, LD_TIME, MODIFY_TIME / DWS: LD_TIME, MODIFY_TIME）
4. 系统字段不重复追加（如果映射中已包含则跳过）

```sql
DROP TABLE IF EXISTS {Schema}.{表名};
CREATE TABLE {Schema}.{表名} (
   {字段1}  {类型} DEFAULT NULL COMMENT '{注释}',
   {字段2}  {类型} DEFAULT NULL COMMENT '{注释}',
   ...
   SSYS  STRING DEFAULT NULL COMMENT '源系统',          -- 仅 DWD
   SRC_TAB  STRING DEFAULT NULL COMMENT '源表',        -- 仅 DWD
   LD_TIME  STRING DEFAULT NULL COMMENT '加载时间',
   MODIFY_TIME  STRING DEFAULT NULL COMMENT '修改时间'
)
COMMENT '{表中文名}'
PARTITIONED BY ( P_DT STRING )
ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t' NULL DEFINED AS '' ;
```

#### 6.2.2 ETL 生成（Shell 脚本）

**文件结构：**
```bash
#!/bin/sh
source ${hadoop_env}
source ${source_path}

inceptor_beeline() {
    beeline -u ${hive_jdbc} -n ${hive_username} -p ${hive_password} -e "$1"
}

inceptor_beeline "

-- 表头注释
-- ** 主题:{主题}
-- ** 描述:{描述}
-- ** 目标表：{Schema}.{表名}

set hive.exec.dynamic.partition=true;
set hive.exec.dynamic.partition.mode=nonstrict;
set hive.exec.max.dynamic.partitions.pernode=10000;
set hive.exec.max.dynamic.partitions=10000;

INSERT OVERWRITE TABLE {Schema}.{表名} PARTITION(P_DT = '${{参数}}')
SELECT
       T1.{字段}                     AS {目标字段}                     --{注释}
     , T2.{字段}                     AS {目标字段}                     --{注释}
     , 'O32'                         AS SSYS                          --源系统
     , '表1,表2'                     AS SRC_TAB                       --源表
     , FROM_UNIXTIME(UNIX_TIMESTAMP(CURRENT_TIMESTAMP()),'yyyy-MM-dd HH:mm:ss') AS LD_TIME   --加载时间
     , FROM_UNIXTIME(UNIX_TIMESTAMP(CURRENT_TIMESTAMP()),'yyyy-MM-dd HH:mm:ss') AS MODIFY_TIME --修改时间
FROM {源表1} T1
  LEFT JOIN {源表2} T2 ON T1.{字段} = T2.{字段}
    AND T2.{字段} = '条件'
WHERE 过滤条件1
  AND 过滤条件2
;"
```

**SELECT 子句对齐规则：**
```python
_AS_POS = 80        # " AS 别名" 起始列
_COMMENT_POS = 120  # "--注释" 起始列

# 格式：前缀 + 表达式.ljust(80) + " AS " + 别名.ljust(120 - 80 - len(' AS 别名')) + "--注释"
#    T1.CUST_ID                                                                    AS CUST_ID                        --客户ID
```

**CASE 表达式格式化：**
```sql
 -- 多行 CASE
 , CASE WHEN T1.C_CUSTTYPE IS NOT NULL
        THEN COALESCE(T3.STD_DICT_CLS_CD_VAL,CONCAT('@',T1.C_CUSTTYPE))
    END                                                                          AS CUST_TYPE_CODE                      --客户类型代码
```

## 11. 使用方式

```bash
# 全流程生成（使用配置中的默认路径，仅 O32）
python3 tools/main.py --layer ALL

  --field-survey "/path/to/字段级调研.xlsx"

python3 tools/main.py --layer DWD \

python3 tools/main.py --layer DWS \
  --dws-mapping "/path/to/DWS/mapping.xlsx"

python3 tools/main.py --layer ODS --output /custom/output/path \
  --table-survey ... --field-survey ...
```

| `--field-survey` | config.FIELD_SURVEY_PATH | 字段级调研 Excel 路径 |
| `--dwd-mapping-dir` | config.DWD_MAPPING_DIR | DWD MAPPING 目录 |

```bash
survey_base="/path/to/01-系统调研文档"
```

### 13.3 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| ODS 生成 0 张表 | 调研文档路径错误或字段映射不匹配 | 检查输入路径和 `is_reserved` 字段 |
| DWD 跳过大量表 | MAPPING 中目标表名含非法字符 | 检查 "装数的表英文名称" 等占位符 |
| 字段类型全是 STRING | 字段级调研中缺少 Oracle 类型 | 补全字段级调研的"字段类型"列 |
| 生成后表名有下划线过多 | 源字段名含空格/特殊字符 | `_sanitize_identifier()` 自动修复 |
| ETL 脚本中 DataX 配置为空 | 模板文件缺失 | 检查 `demo/templates/etl_ods_template.sh` |

### 13.4 错误处理规则

1. **非法标识符**：自动修复（替换非法字符为下划线）或跳过并记录 ERROR 日志
2. **文件 IO 错误**：抛出异常中止（不掩盖 IO 问题）
3. **跳过表**：生成器 batch 方法 catch ValueError，跳过并记录 ERROR，继续处理下一张
4. **Excel 解析错误**：catch 异常，记录 WARNING，跳过当前 sheet

### 13.5 代码质量约束

- 不使用 `print()`，统一使用 `logging` 模块
- 表名/字段名必须通过 `validate_db_identifier()` 验证
- 输出路径必须通过 `validate_output_path()` 验证
- 字段序号 `tgt_ordinal` 使用 `safe_float_str()` 安全转换
- Sheet 数据使用 `safe_str()` 安全取值（NaN → ""）
